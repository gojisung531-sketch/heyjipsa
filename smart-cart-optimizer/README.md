# smart-cart-optimizer

> 한국 쇼핑몰 무료배송 기준에 맞춰 장바구니 묶음을 자동 최적화하는 Cowork 스킬

## 만든 이유

장보기 한 번 할 때마다 머릿속으로 "쿠팡은 19,800원, 컬리는 4만원, SSG는 4만원…"
계산기 두드리면서 어디다 뭘 묶어야 배송비 0원 되는지 짜맞추는 게 너무 피곤했다.

게다가 "이건 브랜드여야 해(컬리)" / "이건 그냥 싼 거(쿠팡)" 처럼
같은 휴지여도 어디서 살지 다른데, 그걸 매번 머리로 굴리는 게 손해보는 기분.

**그냥 품목만 던지면 알아서 묶어주면 좋잖아?** 라는 생각으로 만들었다.

## 어떻게 동작하나

1. 품목 리스트 입력받기 (이름·가격·품질 태그·강제 플랫폼 옵션)
2. 품질 태그 기준으로 1차 플랫폼 배정
   - `lowest` → 쿠팡
   - `brand` + 신선식품 키워드 → 마켓컬리
   - `brand` + 일반 → SSG
   - `any` → 일단 보류
3. 보류된 `any` 품목으로 무배 기준 부족한 플랫폼 채우기 (그리디)
4. 플랫폼별 합계 계산 → 무배 충족 여부 확인
5. 마크다운 테이블 + 총 배송비 + 절약 금액 출력

## 사용 예시

### 입력

```
휴지:any:18000
라면:lowest:7500
우유:brand:4000
계란:brand:7200
그래놀라:brand:30000
```

### 출력

```
# 🛒 묶음 추천 결과

## 쿠팡 — 25,500원 ✅ 무배
| 품목 | 가격 | 태그 |
|------|------|------|
| 라면 | 7,500원 | lowest |
| 휴지 | 18,500원 | any |

## 마켓컬리 — 41,200원 ✅ 무배
| 품목 | 가격 | 태그 |
|------|------|------|
| 우유 | 4,000원 | brand |
| 계란 | 7,200원 | brand |
| 그래놀라 | 30,000원 | brand |

## 총 배송비: 0원
## 💰 절약: 6,000원 (플랫폼별 따로 시켰을 때 대비)
```

## 설치

이 스킬은 Cowork 스킬 형식이라 별도 설치 없이 폴더만 있으면 된다.
Claude에 폴더 마운트하고 "smart-cart-optimizer 써서 묶음 짜줘" 하면 끝.

직접 CLI로 돌리고 싶으면:

```bash
cd smart-cart-optimizer
python cli.py "휴지, 라면:lowest, 우유:brand:4000, 계란:brand:7200"
```

옵션:

- `--file items.txt` : 한 줄당 한 품목 (인라인과 같은 문법)
- `--json items.json` : `[{"name":"휴지","tag":"any","price":18000}, ...]`
- `--db my_platforms.json` : 무배 기준 커스텀 DB

### 인라인 문법

```
이름[:tag][:price][:platform]
```

- `tag`: `brand` / `lowest` / `any`
- `price`: 숫자(원)
- `platform`: `쿠팡` / `마켓컬리` / `SSG` / `네이버쇼핑`

순서 무관. 예시 모두 valid:
- `우유` (가격은 DB에서 추정)
- `우유:brand`
- `우유:brand:4000`
- `우유:brand:4000:마켓컬리`
- `우유:4000:brand` (순서 바꿔도 OK)

## 플랫폼 DB 수정

`platforms.json` 직접 편집하면 됨.

```json
{
  "platforms": {
    "11번가": {
      "free_shipping_threshold": 30000,
      "shipping_fee": 2500,
      "best_for": ["lowest"],
      "note": "수퍼배송 기준"
    }
  }
}
```

기본 가격 사전(`default_item_prices`)도 같은 파일에 있음.
자주 사는 품목 추가해두면 가격 입력 안 해도 자동 추정.

## 한계

- **실시간 가격 X** — 외부 API 안 씀. 사용자 입력 or 내장 DB 추정값.
  실제 시세와 다를 수 있음. 큰 금액 차이 나면 직접 가격 입력 권장.
- **그리디 알고리즘** — 품목 100개 넣어도 돌긴 돌지만 글로벌 최적해 보장 X.
  보통 10~20개 수준에서는 충분히 잘 나옴.
- **네이버쇼핑** — 판매자별 무배 기준이 다 달라서 기본값 `null`. 본인 자주 쓰는 셀러 기준을 직접 입력해야 함.
- **카드 할인·쿠폰·적립** — 계산 안 함. 순수 배송비 최적화만.
- **재고/품절** — 못 잡음. 묶음대로 사려는데 한 품목이 그 플랫폼에 없을 수 있음.
