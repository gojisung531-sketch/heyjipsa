# Smart Cart Optimizer

## What this skill does

여러 쇼핑 품목을 입력받아 각 플랫폼(쿠팡 19,800원 / 마켓컬리·SSG 40,000원 / 네이버쇼핑 사용자 정의)의 무료배송 기준을 맞추도록 묶음 조합을 추천한다. 품목별 품질 태그(`brand`/`lowest`/`any`)에 따라 1차 플랫폼 배정 후, 부족한 무배 기준을 채우기 위해 `any` 태그 품목을 재배치한다.

## When to use

- "이번 주 장보기 한 번에 시킬건데, 어디서 어떻게 묶지?" 류 질문
- 여러 생필품·식료품·상품을 한꺼번에 사야 하는 상황
- "쿠팡 무배 기준 19,800원 맞추려고 뭐 더 살까" 같은 미세조정 질문
- 사용자가 품목 리스트를 5개 이상 던지면서 가격이나 플랫폼을 고민할 때
- "배송비 아까운데" / "묶음배송 어떻게 해?" 같은 표현이 등장할 때

## How to use

1. 사용자에게 품목을 받는다. 형식이 자유로워도 됨:
   - 단순 리스트: `휴지, 세제, 우유, 라면`
   - 태그 포함: `우유:brand, 라면:lowest, 휴지:any`
   - 가격 포함: `세제:any:15000`
   - 플랫폼 강제: `샴푸:brand:18000:마켓컬리`
2. 품목별로 dict 리스트를 만들어 `optimizer.optimize(items)` 호출.
   - 가격이 빠지면 `platforms.json`의 `default_item_prices` 에서 추정. 없으면 0원 처리하고 경고.
3. 반환된 결과를 `format_report()` 로 마크다운 변환해서 보여준다.
4. 사용자가 플랫폼 DB(무배 기준)를 수정하고 싶다면 `platforms.json` 직접 편집을 안내.

## Output format

플랫폼별 묶음 테이블 + 총 배송비 + 절약 금액. 형식:

```
# 🛒 묶음 추천 결과

## 쿠팡 — 23,400원 ✅ 무배
| 품목 | 가격 | 태그 |
|------|------|------|
| 라면 | 7,500원 | lowest |
| 휴지 | 18,500원 | any |

## 마켓컬리 — 41,200원 ✅ 무배
| 품목 | 가격 | 태그 |
|------|------|------|
| 우유 | 4,000원 | brand |
| 계란 | 7,200원 | brand |
| 그래놀라 | 30,000원 | brand |

## 총 배송비: 0원
## 💰 절약: 6,000원 (플랫폼별 따로 시켰을 때 대비)
```

부족분이 있는 플랫폼은 제목에 `(무배까지 X,XXX원 부족)` 표시.

## Examples

**예시 1 - 기본 입력**

입력: `휴지:any:18000, 라면:lowest:7500, 우유:brand:4000, 계란:brand:7200, 그래놀라:brand:30000`

출력:
```
## 쿠팡 — 25,500원 ✅ 무배
- 라면, 휴지

## 마켓컬리 — 41,200원 ✅ 무배
- 우유, 계란, 그래놀라

총 배송비: 0원
절약: 6,000원
```

**예시 2 - 가격 자동 추정**

입력: `휴지, 세제, 우유, 라면`
→ `platforms.json` DB에서 가격 중간값으로 추정 후 묶음 계산.

**예시 3 - 무배 기준 미달**

입력: `우유:brand:4000, 계란:brand:7200`
출력:
```
## 마켓컬리 — 11,200원 ❌ 배송비 3,000원 (무배까지 28,800원 부족)
```
→ 사용자에게 "무배 채우려면 28,800원 더 사야 함. 어떤 품목 추가할까요?" 안내 가능.

## Edge cases

- **빈 입력**: "최소 1개 이상 품목 입력해주세요" 안내
- **가격 불명**: `default_item_prices` DB에 없으면 0원 처리하고 상단에 경고 출력
- **모든 품목이 한 플랫폼에만 강제 배정**: 다른 플랫폼은 결과에서 제외(빈 묶음 출력 안함)
- **무배 기준 null인 플랫폼(네이버쇼핑)**: 무배 계산에서 제외. 사용자가 임의 기준 입력하면 platforms.json 수정 안내
- **품목이 너무 많아 한 플랫폼 합계가 무배 한참 초과**: 그대로 OK. 분할 추천 안 함(분할은 이 스킬의 책임 아님)

## Limitations

- 외부 API 호출 안 함 → 실시간 가격 못 가져옴. 사용자 입력 또는 내장 DB의 추정값 사용.
- 그리디 알고리즘이라 글로벌 최적해 보장 안 함. 품목 수가 적으면 충분히 정확.
- 네이버쇼핑은 판매자별 무배 기준이 다 달라서 기본 DB에서 `null` 처리. 직접 추가 필요.
- 한국 4대 쇼핑몰 기준. 11번가·G마켓·옥션 등은 사용자가 `platforms.json` 에 추가해야 함.
- 적립금·쿠폰·카드 할인은 계산 안 함. 순수 배송비 최적화만.
