# shipping-fee-filter

네이버쇼핑에서 "990원!" 보고 클릭했는데 배송비 3,500원이라서 빡친 적 있나요?
이 스킬은 그런 낚시 상품을 자동으로 걸러줍니다.

## 만든 이유

에이블리에서 양말 990원짜리 보고 좋아서 장바구니 담았는데, 결제하려니까 배송비가 3,500원. 그러면 실질가격 4,490원인데, 정렬은 여전히 "990원순"으로 1위에 떠 있음.

쇼핑몰들이 "최저가순" 정렬이라면서 실제로는 상품가격만 기준으로 줄 세워주니까, 사용자는 매번 클릭 → 배송비 확인 → 뒤로가기를 반복해야 됩니다.

그래서 만들었습니다. 상품 리스트만 던져주면:
- 실질가격(상품가 + 배송비)으로 다시 줄 세우고
- 배송비가 비정상적으로 높은 낚시 상품에 🚨 표시

## 뭐 하는 스킬?

1. 상품 리스트 입력 받기 (텍스트 또는 캡처 이미지)
2. **실질가격 = 상품가 + 배송비**로 재계산
3. 낚시 의심 상품 표시
4. 실질가격 오름차순으로 재정렬한 표 출력

## 낚시 판별 기준

다음 중 **하나라도** 해당되면 🚨 낚시 의심:

- **배송비가 상품가격의 50% 이상** (배송비로 마진 뽑는 패턴)
- **배송비가 입력 리스트 평균 배송비의 2배 이상** (다른 상품 대비 비정상적으로 비쌈)

## 사용법

### 1) Cowork / Claude Desktop에서 자동 호출

상품 리스트를 던지면 Claude가 알아서 이 스킬을 호출합니다.

```
가은: 이 중에 진짜 최저가 뭐야?
A상품 990원, 배송비 3,500원
B상품 3,200원, 무료배송
C상품 2,500원, 배송비 2,500원
```

→ Claude가 표로 정리해서 답변.

### 2) 직접 Python으로 실행

```bash
python filter.py
```

또는 코드에서 import:

```python
from filter import parse_text_input, detect_bait, format_table

text = """A상품, 990, 3500
B상품, 3200, 0
C상품, 2500, 2500"""

products = parse_text_input(text)
results = detect_bait(products)
print(format_table(results))
```

### 입력 형식

한 줄에 하나의 상품, `이름, 상품가, 배송비` 순서.
구분자는 쉼표(`,`), 슬래시(`/`), 파이프(`|`), 탭 모두 OK.

```
A상품, 990, 3500
B상품 / 3200 / 0
C상품 | 2500 | 2500
```

숫자에 "원", "," 같은 문자가 섞여 있어도 자동 제거합니다.

## 출력 예시

```
| 순위 | 상품명 | 상품가 | 배송비 | 실질가격 | 낚시 의심 |
|------|--------|--------|--------|----------|----------|
| 1 | B상품 | 3,200원 | 무료 | 3,200원 | - |
| 2 | C상품 | 2,500원 | 2,500원 | 5,000원 | 🚨 배송비/상품가 100% |
| 3 | A상품 | 990원 | 3,500원 | 4,490원 | 🚨 배송비/상품가 354% |

**실질가격 기준 최저가: B상품 (3,200원)**
```

## 한계

- 평균 배송비는 **입력 리스트 안에서만** 계산합니다. 외부 카테고리 데이터를 안 쓰기 때문에, 같은 카테고리 상품끼리 비교해야 정확해요.
- 이미지 OCR은 Claude의 비전 능력에 의존합니다. 화질이 나쁘면 가격을 잘못 읽을 수 있음.
- 도서산간/지역별 배송비 차이는 반영 안 합니다.
- 묶음배송, 조건부 무료배송(예: 2만원 이상 무료)은 반영 안 합니다.
- 외부 쇼핑몰 API를 호출하지 않습니다 — 사용자가 직접 데이터를 줘야 작동합니다.
