# Shipping Fee Filter

## What this skill does

온라인 쇼핑 최저가 검색에서 상품가격에 배송비를 더한 **실질가격**으로 재정렬하고, 배송비가 비정상적으로 높은 **낚시 상품**을 자동으로 표시한다.

## When to use

- 네이버쇼핑, 에이블리, 쿠팡, 11번가, 지마켓 등 쇼핑몰 비교 시
- 사용자가 상품명/가격/배송비가 들어있는 리스트(텍스트 또는 캡처 이미지)를 올렸을 때
- "이게 진짜 최저가야?", "배송비까지 포함하면 얼마야?", "낚시 아니야?" 같은 질문이 나올 때
- 가격은 낮은데 배송비가 이상하게 비싼 상품을 의심할 때

## How to use

1. **입력 받기**
   - 텍스트라면 그대로 사용
   - 캡처 이미지라면 Claude가 직접 OCR해서 상품명/가격/배송비를 추출 후 텍스트화
   - 텍스트 형식 예: `A상품, 990, 3500` (이름, 상품가, 배송비)
   - 구분자는 쉼표(,), 슬래시(/), 파이프(|), 탭 모두 지원

2. **filter.py 실행**
   ```bash
   python filter.py
   ```
   또는 직접 import해서 사용:
   ```python
   from filter import parse_text_input, detect_bait, format_table
   products = parse_text_input(user_text)
   results = detect_bait(products)
   print(format_table(results))
   ```

3. **낚시 판별 로직**
   - 조건 A: `배송비 / 상품가격 >= 0.5` (배송비가 상품가의 50% 이상)
   - 조건 B: `배송비 >= 입력 리스트의 배송비 평균 × 2`
   - 둘 중 하나라도 해당되면 🚨 낚시 의심

4. **출력**: 실질가격 오름차순으로 정렬한 비교 테이블 + 최저가 한 줄 요약

## Output format

마크다운 표 형식으로 출력. 컬럼: 순위 / 상품명 / 상품가 / 배송비 / 실질가격 / 낚시 의심.

```
| 순위 | 상품명 | 상품가 | 배송비 | 실질가격 | 낚시 의심 |
|------|--------|--------|--------|----------|----------|
| 1 | B상품 | 3,200원 | 무료 | 3,200원 | - |
| 2 | A상품 | 990원 | 3,500원 | 4,490원 | 🚨 배송비/상품가 354% |
```

표 아래에 마지막 줄로 `**실질가격 기준 최저가: [상품명] ([가격]원)**` 한 줄 요약.

## Examples

### 예시 1: 명백한 낚시 상품
**입력:**
```
A상품, 990, 3500
B상품, 3200, 0
```
**출력:**
| 순위 | 상품명 | 상품가 | 배송비 | 실질가격 | 낚시 의심 |
|------|--------|--------|--------|----------|----------|
| 1 | B상품 | 3,200원 | 무료 | 3,200원 | - |
| 2 | A상품 | 990원 | 3,500원 | 4,490원 | 🚨 배송비/상품가 353% |

**실질가격 기준 최저가: B상품 (3,200원)**

### 예시 2: 카테고리 평균 초과형
**입력:**
```
양말A, 2000, 2500
양말B, 2500, 2500
양말C, 1000, 6000
양말D, 3000, 2500
```
**출력:**
| 순위 | 상품명 | 상품가 | 배송비 | 실질가격 | 낚시 의심 |
|------|--------|--------|--------|----------|----------|
| 1 | 양말A | 2,000원 | 2,500원 | 4,500원 | 🚨 배송비/상품가 125% |
| 2 | 양말B | 2,500원 | 2,500원 | 5,000원 | 🚨 배송비/상품가 100% |
| 3 | 양말D | 3,000원 | 2,500원 | 5,500원 | - |
| 4 | 양말C | 1,000원 | 6,000원 | 7,000원 | 🚨 배송비/상품가 600%, 평균 배송비의 1.7배 |

**실질가격 기준 최저가: 양말A (4,500원)**

### 예시 3: 모두 정상
**입력:**

**출력:**

|------|--------|--------|--------|----------|----------|

**실질가격 기준 최저가: 티셔츠1 (15,000원)**

## Edge cases

- **배송비 0원(무료배송)**: 낚시 아님. ratio 계산에서 자동 0.
- **상품가격 0원 이하**: 입력 에러로 간주, 해당 행 스킵.
- **입력 1개**: 카테고리 평균 비교는 무의미하므로 조건 A만 적용.
- **모든 상품 무료배송**: 평균 배송비 0이면 조건 B는 절대 발동 안 함, 정상.
- **파싱 실패한 줄**: 스킵하고 사용자에게 "X줄을 인식하지 못했습니다" 안내.
- **이미지 OCR 실패**: 사용자에게 텍스트로 다시 입력 요청.

## Limitations

- 카테고리 평균은 외부 데이터가 아니라 **입력 리스트 내에서만** 계산. 입력에 한 카테고리 상품만 넣어야 정확함.
- 이미지 OCR은 Claude의 비전 능력에 의존, 화질이 나쁘면 가격 오인식 가능.
- 지역별/도서산간 배송비 차이는 반영 안 함.
- 묶음배송, 조건부 무료배송(예: 2만원 이상 무료) 같은 복합 정책은 반영 안 함.
- 외부 쇼핑몰 API 호출 안 함 — 사용자가 직접 데이터를 줘야 함.
