# chore-logger

자연어로 "누가 뭐 했어" 한 줄 던지면, 누적 CSV 로그에 자동으로 구조화해서 적어주는 Cowork 스킬.

```
입력:  "오늘 내가 설거지 하고 빨래 돌림"
결과:  chore_log.csv에 2줄 추가
       2026-05-06 | 나 | 설거지 | 20분
       2026-05-06 | 나 | 빨래   | 10분
```

## 만든 이유

집안일 분담 가지고 가족끼리 어색해지는 게 매번 똑같은 패턴이었음.
"내가 더 많이 하잖아" vs "나도 했거든" — 둘 다 진짜인데 데이터가 없어서 평행선.

근데 매번 노션에 표 만들어서 적자고 하면 누구도 안 함. 진입장벽이 너무 높음.
대신 "엄마 오늘 빨래 돌렸어"처럼 평소 말투 그대로 한 줄 던지면 자동으로 기록되면 어떨까 해서 만듦.

한 달치 쌓이면 누가 뭘 얼마나 했는지 진짜 데이터로 보임. 감정 싸움이 통계 싸움이 됨 (이게 더 나음).

## 설치

레포 클론하거나 SKILL.md + scripts/ 폴더만 Cowork 스킬 디렉터리에 넣으면 됨.

```bash
git clone https://github.com/<USERNAME>/K-Skills.git
```

Python 3.8+ 필요. 외부 라이브러리 없음 (표준 라이브러리만 사용).

## 사용법

Cowork에서 그냥 자연어로 말하면 자동 트리거됨:

```
"오늘 내가 설거지 했어"
"엄마가 1시간 청소함"
"어제 아빠가 분리수거 했어"
```

직접 CLI로 돌리려면:

```bash
python scripts/log_chore.py "오늘 내가 설거지 하고 빨래 돌림"
```

미리 결과만 확인 (기록 안 함):

```bash
python scripts/log_chore.py "엄마가 30분 청소함" --dry-run
```

## 가족 설정

처음 실행하면 현재 디렉터리에 `family.json`이 자동 생성됨.
기본값은 `나 / 엄마 / 아빠`. 자기 가족에 맞게 수정:

```json
{
  "self_name": "가은",
  "self_aliases": ["나", "내가", "저", "제가"],
  "members": [
    {"name": "엄마", "aliases": ["엄마", "어머니", "맘"]},
    {"name": "아빠", "aliases": ["아빠", "아버지"]},
    {"name": "동생", "aliases": ["동생", "지호"]}
  ]
}
```

`self_aliases`는 1인칭 표현. 사용자가 직접 입력할 때 자기 자신을 가리키는 단어.
`members`는 가족 구성원. 별칭 여러 개 등록 가능.

## 카테고리와 기본 소요시간

| 카테고리 | 키워드 (일부) | 기본 소요시간 |
|---------|-------------|--------------|
| 설거지 | 설거지, 그릇 닦, 접시 닦 | 20분 |
| 빨래 | 빨래, 세탁, 건조기 | 10분 |
| 청소 | 청소, 청소기, 걸레질, 바닥 닦 | 30분 |
| 요리 | 요리, 밥 했, 저녁 했, 음식 | 40분 |
| 쓰레기 | 쓰레기, 분리수거, 음쓰, 음식물 | 5분 |
| 장보기 | 장보, 장 봐, 마트, 쇼핑 | 60분 |
| 기타 | (위 어디도 안 걸리면) | 15분 |

문장에 "30분", "1시간 20분"처럼 시간이 명시되어 있으면 그게 우선.

## 로그 파일

기본 위치: 현재 디렉터리의 `chore_log.csv`.
스프레드시트로 열거나 pandas로 분석 가능:

```python
import pandas as pd
df = pd.read_csv("chore_log.csv")
df.groupby("person")["duration_min"].sum()
```

다른 경로로 저장하려면:

```bash
python scripts/log_chore.py "..." --log ~/Documents/our_chores.csv
```

## 한계

- 한국어 발화 전용. 영어는 카테고리 매칭 안 됨.
- 가족 구성원은 `family.json`에 직접 등록한 별칭만 인식. AI 추론 안 함.
- "저녁 7시쯤", "퇴근하고 나서" 같은 자연어 시각은 현재 시각으로 처리.
- 한 문장에 두 사람이 다 등장하면 매칭이 모호해짐 (두 문장으로 나눠 입력 권장).
- 외부 API 의존 없음 → 차단 우려 없음. 오프라인 동작.
