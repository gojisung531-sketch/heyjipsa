# Chore Logger

## What this skill does

자연어로 입력된 집안일 문장을 파싱해서 `누가 / 뭘 / 언제 / 얼마나` 4개 필드로 구조화하고, 누적 로그 파일(`chore_log.csv`)에 한 줄(또는 여러 줄)로 append한다. 한 문장에 집안일이 여러 개 들어 있으면 각각 별도 row로 기록한다.

## When to use

- 사용자가 "오늘 내가 설거지 했어", "엄마가 빨래 돌림" 같이 **누가 뭘 했다**는 형식의 일상 발화를 할 때
- "집안일 기록", "가사노동 추적", "이번 주 누가 뭐 얼마나 했는지" 같은 표현이 나올 때
- 한 문장에 여러 집안일이 섞여 있을 때 (자동 분리)
- 명시적 시간이 없으면 "지금"으로, 명시적 소요시간이 없으면 카테고리 기본값으로 채움

다음 경우엔 **사용하지 않음**:
- 단순히 집안일 카테고리 설명을 묻는 질문 (기록 의도가 없음)
- 일정/할일 등록 (그건 task-management 스킬 영역)

## How to use

1. 사용자 발화를 그대로 1번째 인자로 넘긴다.
2. 스크립트 실행:
   ```bash
   python scripts/log_chore.py "오늘 내가 설거지 하고 빨래 돌림"
   ```
3. 처음 실행 시 현재 디렉터리에 `family.json`이 자동 생성된다 (기본 가족 구성: 나/엄마/아빠). 사용자가 자기 가족 이름·별칭으로 수정.
4. 로그는 현재 디렉터리의 `chore_log.csv`에 append된다.
5. 미리 결과만 보고 싶으면 `--dry-run` 추가:
   ```bash
   python scripts/log_chore.py "엄마가 30분 청소함" --dry-run
   ```

### 옵션

- `--log <경로>`: 로그 파일 경로 변경 (기본값: `./chore_log.csv`)
- `--family <경로>`: 가족 설정 파일 경로 변경 (기본값: `./family.json`)
- `--dry-run`: 파싱 결과를 JSON으로 출력하고 종료 (기록 안 함)

## Output format

CSV 컬럼 순서 고정:

| date | time | person | category | duration_min | raw_input |
|------|------|--------|----------|--------------|-----------|
| 2026-05-06 | 19:30 | 나 | 설거지 | 20 | 오늘 내가 설거지 하고 빨래 돌림 |
| 2026-05-06 | 19:30 | 나 | 빨래 | 10 | 오늘 내가 설거지 하고 빨래 돌림 |

콘솔 출력은 사람이 읽기 쉬운 요약:
```
기록 완료: 2건
  - 2026-05-06 19:30 | 나 | 설거지 | 20분
  - 2026-05-06 19:30 | 나 | 빨래 | 10분
```

## Examples

### 예시 1: 1인칭 + 복수 집안일

입력:
```
오늘 내가 설거지 하고 빨래 돌림
```

출력 (CSV에 append되는 row):
```
2026-05-06,19:30,나,설거지,20,오늘 내가 설거지 하고 빨래 돌림
2026-05-06,19:30,나,빨래,10,오늘 내가 설거지 하고 빨래 돌림
```

### 예시 2: 가족 구성원 + 명시적 소요시간

입력:
```
엄마가 1시간 청소함
```

출력:
```
2026-05-06,19:30,엄마,청소,60,엄마가 1시간 청소함
```

### 예시 3: 어제 + 단일 작업

입력:
```
어제 아빠가 분리수거 했어
```

출력:
```
2026-05-05,19:30,아빠,쓰레기,5,어제 아빠가 분리수거 했어
```

## Edge cases

- **담당자 못 찾음**: `family.json`에 없는 이름이면 기본값 `self_name`("나")으로 기록. 사용자에게 가족 설정 수정하라고 안내.
- **카테고리 못 찾음**: 어떤 카테고리 키워드도 없으면 `기타` (기본 15분)로 기록.
- **빈 입력**: 종료 코드 1로 실패. "입력 없음" 에러 메시지.
- **소요시간 충돌**: 텍스트에 "30분"이 명시되어 있으면 그게 우선, 없으면 카테고리 기본값.
- **여러 카테고리 동시 매칭**: 한 작업이 여러 카테고리 키워드에 걸리면 첫 번째 매칭 카테고리만 사용 (중복 방지).
- **날짜 키워드**: "어제", "그제/그저께"만 인식. 그 외에는 오늘로 처리.

## Limitations

- 한국어 발화에 최적화. 영어 입력은 카테고리 키워드 매칭이 안 될 수 있음.
- 가족 구성원 별칭은 `family.json`에 직접 등록한 것만 인식. AI 추론 X.
- 시간 파싱은 단순 규칙 기반. "저녁 7시쯤", "퇴근하고" 같은 자연어 시각 표현은 인식 X (현재 시각으로 기록).
- "내가 했고 엄마가 빨래" 같이 한 문장에 담당자가 둘인 경우 가족 멤버 우선 매칭이라 마지막 멤버 한 명에게 모든 작업이 귀속될 수 있음. 두 사람이 동시에 한 일은 문장을 나눠 입력 권장.
