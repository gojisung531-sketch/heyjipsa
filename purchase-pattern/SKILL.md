# Purchase Pattern Analyzer

## What this skill does

여러 개의 구매 기록 파일(xlsx, csv, md)을 읽어 품목별로 묶고, 평균 구매 주기 / 평균 단가 / 총 지출 / 다음 구매 예상일을 계산한다. 결과는 정리된 표(콘솔용 마크다운)와 인터랙티브 HTML 리포트(Chart.js 막대·도넛 그래프 + 정렬 가능한 표)로 출력한다.

## When to use

- 사용자가 영수증 / 가계부 / 구매 기록 파일을 여러 개 모아놓고 분석을 요청할 때
- "내가 휴지 얼마나 자주 사지", "다음 세제 살 때가 언제일까" 같은 주기 예측 발화
- 월 평균 지출 / 카테고리별 지출 / 자주 사는 물건 TOP N을 알고 싶을 때
- receipt-scanner 스킬의 출력물(누적된 영수증 데이터)을 후처리할 때

## How to use

1. 사용자에게 입력 파일 경로(들) 확인. 폴더 통째로 받는 것도 가능.
2. 품목명 컬럼 / 날짜 컬럼 / 가격 컬럼 / 수량 컬럼 매핑 확인 (xlsx/csv 헤더가 표준이 아니면 AskUserQuestion으로 묻기). 마크다운은 표 형식 가정.
3. `scripts/analyze.py` 실행:
   ```bash
   python scripts/analyze.py --input <파일1> <파일2> ... --output report.html
   ```
   또는 폴더 단위:
   ```bash
   python scripts/analyze.py --input-dir ./receipts --output report.html
   ```
4. 옵션:
   - `--min-count N` : N회 이상 구매한 품목만 (기본 2)
   - `--top N` : 상위 N개만 표시 (기본 30)
   - `--name-col`, `--date-col`, `--price-col`, `--qty-col` : 컬럼명 매핑
5. 결과 HTML과 표 요약을 사용자에게 제시. 다음 구매 예상일 임박한 항목은 강조.

## Output format

**1) 콘솔/채팅 응답 - 마크다운 표:**

| 품목 | 구매 횟수 | 평균 주기 | 평균 단가 | 총 지출 | 다음 예상일 |
|---|---|---|---|---|---|
| 휴지 (코스트코 30롤) | 5 | 3.5주 | 12,900원 | 64,500원 | 2026-05-21 |
| 우유 1L | 12 | 6일 | 3,200원 | 38,400원 | 2026-05-09 |

**2) HTML 리포트 (`report.html`):**

- 상단 KPI: 총 지출 / 분석 기간 / 분석된 거래 건수 / 고유 품목 수
- 차트 1: 월별 총 지출 막대그래프 (Chart.js)
- 차트 2: 카테고리별/품목별 지출 비중 도넛
- 표: 품목별 정렬 가능한 표 (구매 횟수 / 평균 주기(일) / 평균 단가 / 총 지출 / 다음 예상일)
- 다음 예상일 ≤ 7일 이내 행은 노란색, 이미 지났으면 빨간색 배경으로 강조

## Examples

**예시 1 - 단일 csv 파일**

입력 파일 `receipts.csv`:
```
date,item,price,quantity
2026-01-05,휴지 30롤,12900,1
2026-01-30,휴지 30롤,12900,1
2026-02-26,휴지 30롤,12900,1
2026-03-22,휴지 30롤,13500,1
2026-04-19,휴지 30롤,12900,1
```

출력 표 (일부):
```
| 품목 | 횟수 | 평균 주기 | 평균 단가 | 다음 예상일 |
| 휴지 30롤 | 5 | 26일 (3.7주) | 13,020원 | 2026-05-15 |
```

**예시 2 - 여러 파일 합치기**

입력: `2026-01.xlsx`, `2026-02.xlsx`, `2026-03.md` (마크다운 표 형식)

마크다운 예:
```
| 날짜 | 품목 | 가격 | 수량 |
|---|---|---|---|
| 2026-03-02 | 우유 1L | 3200 | 1 |
| 2026-03-08 | 우유 1L | 3200 | 1 |
```

출력 HTML 리포트 + 채팅 요약:
"3개 파일에서 총 47건의 구매를 찾았어. 자주 사는 TOP 3은 우유(12회, 6일 주기), 바나나(8회, 9일 주기), 휴지(5회, 26일 주기). 우유는 다음 예상일 2026-05-09으로 임박."

**예시 3 - 품목명 정규화 필요**

입력에 "코카콜라 1.5L", "코카콜라1.5리터", "Coca-Cola 1.5L"이 섞여 있을 때:

스크립트는 기본적으로 공백·대소문자·단위 표기 차이를 정규화해서 같은 품목으로 묶는다. 다른 품목으로 분리하고 싶으면 `--no-normalize` 옵션 사용.

## Edge cases

- **파일에 헤더 없음**: 사용자에게 컬럼 위치 묻기. 기본 가정은 `date, item, price, quantity` 순서.
- **날짜 형식 제각각** (`2026-01-05`, `26.1.5`, `1월 5일`): pandas의 `to_datetime(errors='coerce')`로 파싱하고, 실패한 행은 제외하면서 콘솔에 경고.
- **단일 구매 품목** (1회만 산 것): 주기 계산 불가 → "1회 구매" 표기, 다음 예상일 빈칸.
- **수량 컬럼 없음**: 모두 1로 가정.
- **품목명 비어있는 행**: 스킵하고 카운트만 보고.
- **너무 적은 데이터** (전체 10건 미만): "데이터가 적어 패턴 신뢰도 낮음" 경고를 HTML 상단에 표시.
- **빈 입력 파일**: 에러 대신 "분석할 데이터가 없습니다" 메시지로 종료.

## Limitations

- 외부 API 의존 없음. pandas + jinja2 + Chart.js(CDN)만 사용. Chart.js는 CDN 차단 환경에서 안 보일 수 있음 → README에 명시.
- 다음 구매 예상일은 **단순 평균 주기 기반**(마지막 구매일 + 평균 간격). 계절성·이벤트(명절·세일) 반영 안 함.
- 품목명 정규화는 휴리스틱(소문자화, 공백·특수문자 제거, 단위 표기 통일). "삼다수 2L"와 "에비앙 2L"는 다른 품목으로 분리됨(브랜드 단위).
- 카테고리 분류 자동화는 안 함. 카테고리 컬럼이 입력에 있으면 사용, 없으면 카테고리 차트는 생략.
- 마크다운 입력은 GitHub 표 형식만 지원. 자유 텍스트 영수증은 receipt-scanner 스킬로 먼저 정형화 필요.
