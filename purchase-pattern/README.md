# Purchase Pattern Analyzer

쌓아둔 영수증/가계부 데이터를 분석해서 **자주 사는 물건, 평균 구매 주기, 평균 단가, 다음 구매 예상일**을 정리해주는 K-Skills 스킬.

receipt-scanner로 모은 데이터든 직접 적은 가계부든, csv·xlsx·markdown 파일을 던져주면 품목별로 묶어서 표 + 인터랙티브 HTML 리포트로 뽑아준다.

## 만든 이유

영수증 사진 찍어서 데이터로 만드는 건 receipt-scanner가 해주는데, 그렇게 쌓인 데이터를 **"그래서 내가 뭘 자주 사는데? 휴지 다 떨어져 가는데 언제 사야 하지?"** 같은 질문으로 바로 연결해주는 도구가 없었다.

엑셀 피벗 돌려서 직접 보는 것도 가능하지만:

- 같은 휴지인데 "30롤", "30 롤", "휴지30롤"로 입력되면 따로 잡힘
- 평균 주기 계산하려면 직접 LAG 함수 같은 거 써야 함
- 다음 예상일까지 자동으로 안 나옴

그래서 그냥 한 방에 처리하는 스킬로 만들었다. 한국 사용자 기준 (원화, 한글 품목명, 한국 마트/편의점) 으로 동작한다.

## 무엇을 하는가

입력: 구매 기록 파일 1개 이상 (xlsx / csv / markdown 표)

처리:
1. 파일 다 읽어서 한 테이블로 합치기
2. 품목명 정규화 (공백·대소문자·단위 표기 통일) → 같은 물건끼리 묶기
3. 품목별로 평균 구매 주기, 평균 단가, 총 지출, 마지막 구매일 계산
4. 마지막 구매일 + 평균 주기 = 다음 구매 예상일

출력:
- **마크다운 표**: 채팅에 바로 표시할 수 있는 정리표
- **HTML 리포트**: Chart.js로 월별 지출 그래프 + 품목별 도넛 + 정렬 가능한 표

## 빠른 사용

```bash
# 폴더 통째로
python scripts/analyze.py --input-dir ./receipts --output report.html

# 파일 여러 개
python scripts/analyze.py --input 2026-01.csv 2026-02.xlsx 2026-03.md --output report.html

# 컬럼명이 표준이 아닐 때
python scripts/analyze.py --input data.csv \
  --name-col 상품명 --date-col 구매일 --price-col 가격 --qty-col 수량
```

옵션:

| 옵션 | 의미 | 기본값 |
|---|---|---|
| `--input` | 분석할 파일들 | 없음 |
| `--input-dir` | 분석할 폴더 (재귀 X) | 없음 |
| `--output` | HTML 리포트 저장 경로 | `report.html` |
| `--min-count` | 최소 구매 횟수 (이하면 제외) | 2 |
| `--top` | 상위 N개만 표시 | 30 |
| `--no-normalize` | 품목명 정규화 끄기 | off |
| `--name-col`, `--date-col`, `--price-col`, `--qty-col` | 컬럼명 직접 지정 | 자동 감지 |

## 입력 파일 포맷

다음 컬럼이 있으면 자동 인식 (한글/영문 둘 다 OK):

- 날짜: `date`, `날짜`, `구매일`, `일자`, `purchased_at`
- 품목: `item`, `name`, `품목`, `상품명`, `product`
- 가격: `price`, `가격`, `금액`, `amount`, `total`
- 수량 (없어도 됨, 1로 가정): `qty`, `quantity`, `수량`, `개수`

마크다운은 GitHub 표 형식이어야 한다:

```markdown
| 날짜 | 품목 | 가격 | 수량 |
|---|---|---|---|
| 2026-03-02 | 우유 1L | 3200 | 1 |
```

## 의존성

- Python 3.9+
- `pandas`
- `openpyxl` (xlsx 읽기)
- `jinja2` (HTML 템플릿)

설치:

```bash
pip install pandas openpyxl jinja2
```

Chart.js는 CDN(`https://cdn.jsdelivr.net/npm/chart.js`)에서 로드한다. **CDN 차단 환경(일부 사내망)에서는 차트가 안 보일 수 있음.** 표는 정상 표시.

## 예시

`sample_data/receipts.csv` 같은 데이터:

```csv
date,item,price,quantity
2026-01-05,휴지 30롤,12900,1
2026-01-30,휴지 30롤,12900,1
2026-02-26,휴지 30롤,12900,1
2026-03-22,휴지 30롤,13500,1
2026-04-19,휴지 30롤,12900,1
2026-01-08,우유 1L,3200,1
2026-01-14,우유 1L,3200,2
2026-01-21,우유 1L,3200,1
```

실행:

```bash
python scripts/analyze.py --input sample_data/receipts.csv --output report.html
```

채팅 응답 (예시):

```
| 품목 | 횟수 | 평균 주기 | 평균 단가 | 총 지출 | 다음 예상일 |
|---|---|---|---|---|---|
| 휴지 30롤 | 5 | 26일 (3.7주) | 13,020원 | 65,100원 | 2026-05-15 |
| 우유 1L | 3 | 6일 | 3,200원 | 12,800원 | 2026-01-27 |
```

## 한계

- **다음 예상일은 단순 평균 기반.** 명절·세일·여행 같은 이벤트 반영 안 함.
- **카테고리 자동 분류 안 함.** "휴지", "세제"가 같은 "생필품"인지는 모름. 카테고리 컬럼이 입력에 있으면 그걸 씀.
- **품목명 정규화는 휴리스틱.** "삼다수 2L"와 "에비앙 2L"는 다른 품목으로 분리(브랜드 단위 인식). 같이 묶고 싶으면 입력 파일에서 미리 정리하거나 `--no-normalize` 끈 뒤 후처리.
- **자유 텍스트 영수증(OCR 원본)은 못 읽음.** 미리 표 형식으로 정리되어야 함. receipt-scanner 같은 스킬로 먼저 정형화하기.
- **단일 사용자 가계부 기준.** 가족 공유 가계부에서 "누가 샀는지" 추적은 안 함.
