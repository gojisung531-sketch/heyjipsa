# Budget Guard

## What this skill does
사용자가 붙여넣은 장바구니 텍스트를 파싱해 각 품목을 생필품(✅) / 준생필품(⚠️) / 사치품(🚨)으로 분류하고, 사치품·반복 구매·월 예산 초과 시 경고와 함께 카테고리별 비율 요약을 출력한다.

## When to use
- 사용자가 장바구니/구매 목록을 붙여넣고 검토를 요청할 때
- "이거 사도 될까", "예산 초과인지 체크해줘", "충동구매인지 봐줘" 같은 발화가 나올 때
- "장바구니 정리", "쇼핑 리스트 분류", "사치품 골라줘" 같은 요청
- 월 예산이 명시되고 그 안에서 결제할지 판단해야 할 때

## How to use
1. 사용자에게 장바구니 텍스트를 받는다. 한 줄에 한 품목, `품목명 가격원` 형식 권장 (예: `샤넬 립스틱 52,000원`).
2. 선택 사항으로 월 예산(`monthly_budget`)과 이번 달 누적 지출(`current_spent`)을 받는다.
3. 선택 사항으로 이번 달 구매 이력(`history`, `{"품목명": 횟수}` 형태 dict)을 받는다.
4. `classifier.py` 의 `analyze_cart(text, monthly_budget, current_spent, history)` 를 호출한다.
5. 결과를 `format_report(result)` 에 넘겨 마크다운 리포트로 변환해 그대로 사용자에게 보여준다.

CLI로 빠르게 돌릴 때:
```bash
python classifier.py < cart.txt
```

## Output format
출력은 마크다운으로 다음 5개 블록을 순서대로 포함한다.

1. **품목 분류 테이블** — `| 품목 | 가격 | 분류 |` 헤더, 각 행은 `✅/⚠️/🚨 + 카테고리명`
2. **카테고리별 요약** — 각 카테고리의 개수/금액/비율 + 텍스트 막대 그래프(`█` 5%당 1칸)
3. **총 합계** — 굵은 글씨로 한 줄
4. **경고 섹션** — 사치품 경고 / 반복 구매 경고 / 예산 경고 (해당하는 것만)
5. **반복 구매 경고 문구** — 정확히 `"X 이번 달 N번째 구매예요. 정말 필요한가요?"` 포맷

## Examples

### 예시 1: 기본 케이스
입력:
```
휴지 24롤 18,000원
샤넬 립스틱 52,000원
라면 5개 4,500원
```
출력 (핵심):
```
| 품목 | 가격 | 분류 |
|------|------|------|
| 휴지 24롤 | 18,000원 | ✅ 생필품 |
| 샤넬 립스틱 | 52,000원 | 🚨 사치품 |
| 라면 5개 | 4,500원 | ✅ 생필품 |

🚨 사치품 경고
- 샤넬 립스틱 (52,000원) — 정말 필요한가요?
```

### 예시 2: 예산 초과
입력 (월 예산 30만원, 이미 18만원 지출):
```
디올 쿠션 75,000원
세제 8,000원
```
출력 (핵심):
```
🚨 예산 초과! 월 예산 300,000원 / 예상 지출 263,000원 (88%)
🚨 사치품 경고
- 디올 쿠션 (75,000원) — 정말 필요한가요?
```

### 예시 3: 반복 구매
입력 (이번 달 마카롱 이미 2번 구매):
```
마카롱 6개 15,000원
우유 1L 3,200원
```
출력 (핵심):
```
⚠️ 반복 구매 경고
- '마카롱 6개' 이번 달 3번째 구매예요. 정말 필요한가요?
```

## Edge cases
- **빈 입력**: `{"error": "장바구니에서 품목을 찾을 수 없어요. 형식을 확인해주세요."}` 반환
- **가격 미기재**: 가격 0원으로 처리, 분류만 수행 (총합/비율에서 0 처리)
- **분류 불가능한 품목**: 기본값 `준생필품` (⚠️ 적정량 체크) — 무조건 ✅로 떨어지지 않도록 보수적으로 설정
- **영어/한국어 혼용**: 키워드 사전이 양쪽을 모두 포함 (예: `샤넬` / `chanel`)
- **여러 줄 형식 변형**: `1. 품목 - 가격원`, `품목, 가격`, `품목 가격` 모두 파싱 가능

## Limitations
- 키워드 기반 분류라 신상품/특이 품목은 놓칠 수 있음 → `classifier.py` 의 `CATEGORIES` dict 확장 필요
- 반복 구매 이력은 외부에서 `history` 인자로 주입해야 함 (이 스킬은 영속 저장 안 함)
- 예산 초과 판단은 단순 합산 기준 (할인/쿠폰/적립금 미반영)
- 외부 API 의존 없음 → 차단 가능성 없음
