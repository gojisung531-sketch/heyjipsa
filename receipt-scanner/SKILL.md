# Receipt Scanner

## What this skill does
영수증 이미지를 Claude 멀티모달 OCR로 읽어서 구조화된 표 데이터(날짜/매장명/품목/수량/단가/합계 + 카테고리)로 변환하고, 마크다운 표와 xlsx 파일로 출력한다. 여러 장을 처리할 경우 같은 xlsx 파일에 날짜별로 누적한다.

## When to use
- 사용자가 영수증 이미지(.jpg, .jpeg, .png, .webp, .heic 등)를 첨부했을 때
- "영수증 정리해줘", "영수증 엑셀로 만들어줘", "가계부 입력용으로 정리" 같은 요청이 있을 때
- 여러 장의 영수증을 한 파일로 누적하고 싶다는 요청이 있을 때
- 카테고리별 지출 분류가 필요한 경우

## How to use

### 1단계: 이미지 확인
첨부된 영수증 이미지를 Claude의 비전 능력으로 직접 읽는다. 별도 OCR 라이브러리(Tesseract 등) 호출하지 말 것. Claude가 이미지를 보고 텍스트를 추출하는 게 더 정확함.

### 2단계: 필드 추출
각 영수증에서 아래 필드를 추출:
- **날짜**: YYYY-MM-DD 형식. 영수증에 시간이 있어도 날짜만.
- **매장명**: 영수증 상단의 사업자명/매장명. (예: "GS25 강남역점", "이마트 트레이더스")
- **품목명**: 상품명 그대로. 약어로 찍혀있어도 영수증 표기 그대로 보존.
- **수량**: 숫자. 표기 없으면 1.
- **단가**: 숫자. 콤마/원 표기 제거.
- **합계**: 숫자. 수량 × 단가와 다를 경우(할인 적용 등) 영수증의 실제 합계 우선.

### 3단계: 카테고리 분류
각 품목을 다음 4개 중 하나로 분류:
- **식료품**: 신선식품, 가공식품, 음료, 주류, 과자, 즉석식품 등 먹는 것 전반
- **생필품**: 화장지, 세제, 샴푸, 치약, 비누, 기저귀 등 소모품
- **생활용품**: 주방용품, 문구, 의류, 잡화, 전자제품 등 비소모성 물건
- **기타**: 위 셋에 명확히 안 들어가는 것 (담배, 복권, 서비스 요금 등)

분류 애매하면 매장 맥락 활용. (예: 다이소에서 산 작은 플라스틱 통 → 생활용품)

### 4단계: 마크다운 표 출력
응답에 즉시 표를 보여줌. 형식:

```markdown
## 영수증 (매장명, 날짜)

| 품목 | 수량 | 단가 | 합계 | 카테고리 |
|---|---|---|---|---|
| ... | ... | ... | ... | ... |

**총액: XX,XXX원**
```

### 5단계: xlsx 파일 생성
`scripts/receipt_to_xlsx.py` 실행해서 xlsx 생성.

- **신규 생성**: 사용자가 첫 영수증을 처리하거나 새 파일을 원할 때 → 새 xlsx 생성
- **누적 모드**: 사용자가 "기존 파일에 추가", "이어서 정리" 등을 요청하거나 이미 receipts.xlsx가 워크스페이스에 있으면 → 기존 파일에 행 추가
- 시트는 단일 시트 "Receipts" 사용. 컬럼: `날짜 | 매장명 | 품목 | 수량 | 단가 | 합계 | 카테고리`
- 날짜 기준 오름차순 정렬

## Output format

### 채팅 출력
1. 마크다운 표 (영수증별로 섹션 분리)
2. 영수증별 총액
3. 카테고리별 소계 (영수증이 여러 장이거나 품목이 5개 이상일 때)
4. xlsx 파일 링크 (computer:// 형식)

### xlsx 출력
- 파일명: `receipts.xlsx` (기본) 또는 사용자 지정명
- 위치: 워크스페이스 폴더
- 헤더 행 굵게, 합계 컬럼은 우측 정렬, 통화 형식

## Examples

### 예시 1: 편의점 영수증 1장
입력: GS25 영수증 사진 1장
출력:
```
## GS25 강남역점 (2026-05-06)

| 품목 | 수량 | 단가 | 합계 | 카테고리 |
|---|---|---|---|---|
| 삼각김밥 참치마요 | 2 | 1,500 | 3,000 | 식료품 |
| 포카리스웨트 620ml | 1 | 2,400 | 2,400 | 식료품 |
| 종량제봉투 20L | 1 | 580 | 580 | 생필품 |

**총액: 5,980원**
```
+ receipts.xlsx 파일 링크

### 예시 2: 마트 영수증 (할인 포함)
입력: 이마트 영수증, 일부 품목 할인 적용
출력: 영수증 표기 합계 그대로 사용. 단가 × 수량과 안 맞으면 합계를 우선하고 비고는 생략(데이터 단순화).

### 예시 3: 여러 장 누적
입력: "이번 주 영수증 3장 정리해줘" + 이미지 3장
출력:
- 각 영수증별 마크다운 표 3개
- 카테고리별 소계 표
- 단일 receipts.xlsx 파일 (날짜순 정렬, 모든 품목 누적)

## Edge cases

- **이미지가 흐리거나 일부 글자 식별 불가**: 식별 가능한 부분만 추출하고, 추출 실패한 필드는 `?` 또는 빈 셀로 표시. 사용자에게 어떤 부분이 불명확한지 알림.
- **날짜 형식이 영수증마다 다름**: YYYY-MM-DD로 통일.
- **금액에 콤마/원/₩ 등 단위 포함**: 모두 제거하고 정수로 저장.
- **할인/적립/카드사 정보**: 품목 행으로 취급하지 않음. 합계 계산에 반영된 채로 그대로 둠.
- **영수증이 아닌 이미지**: "이건 영수증이 아닌 것 같아요" 메시지 출력하고 처리 중단.
- **빈 입력 / 이미지 없음**: 사용자에게 영수증 이미지 첨부 요청.
- **기존 xlsx 파일이 손상됐거나 컬럼이 다름**: 백업 파일 만들고(receipts_backup_YYYYMMDD.xlsx) 새 파일로 덮어쓰지 말고 사용자에게 알림.

## Limitations

- 손글씨 영수증, 감열지가 너무 바랜 영수증, 구겨진 영수증은 정확도 낮음.
- 한국 영수증 기준 최적화. 해외 영수증은 카테고리 분류 정확도 떨어질 수 있음.
- 카테고리 분류는 휴리스틱 기반이라 완벽하지 않음. 사용자가 직접 수정해도 되도록 xlsx에 카테고리 컬럼은 자유 편집 가능.
- 영수증에 표기되지 않은 정보(매장 위치, 결제 수단 등)는 추출하지 않음.
- 한 번에 처리하는 이미지 수는 Claude 컨텍스트 제한에 따라 제약됨. 매우 많은 양은 분할 처리 권장.
