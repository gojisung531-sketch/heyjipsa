# Home Tips QA

## What this skill does

한국 살림 팁을 카테고리별로 모아둔 마크다운 DB(`tips_db.md`)를 검색해서 사용자 질문에 답하고, 사용자가 새로 알려준 팁을 같은 DB에 누적 저장한다. 검색은 키워드 매칭(Python), 답변 보강은 Claude가 한다.

## When to use

- 사용자가 살림·생활 관련 질문을 함 ("냄비 탄거 어떻게 닦지?", "수건 냄새 빼는 법", "양파 보관법")
- 사용자가 카테고리 키워드로 팁 요청 ("청소 꿀팁", "절약 팁 알려줘")
- 사용자가 새 팁을 공유함 ("이거 좋더라:", "이런 팁 있던데", "방금 써봤는데", "기억해둬")
- "팁 저장", "팁 추가", "팁 검색" 같은 직접 명령

쇼핑·구매 추천, 의학 상담, 법률 자문에는 사용하지 말 것.

## How to use

### 질문에 답할 때

1. `scripts/search_tips.py "검색어"` 실행 → 관련 팁 후보 반환
2. 결과가 있으면: 가장 관련성 높은 팁 1~3개를 보여주고, 출처가 있으면 같이 표시
3. Claude가 한 줄 보강 (왜 효과 있는지, 주의점 등)
4. 결과가 없으면: 일반 지식으로 답변하되, "DB에 저장된 팁은 없음" 명시

### 팁을 추가할 때

1. 사용자 발화에서 카테고리/내용/출처(있으면)를 파싱
2. 카테고리가 모호하면 한 번만 물어봄 (옵션 6개 제시)
3. `scripts/add_tip.py --category "..." --tip "..." --source "..."` 실행
4. 저장 완료 메시지 + 저장된 위치(`tips_db.md`) 표시

## Output format

### 검색 응답 형식

```
[카테고리] 팁 제목
- 내용: ...
- 출처: (있으면 표시)

💡 보강: Claude의 한 줄 추가 설명
```

여러 개 매칭되면 위 블록을 2~3개까지 나열.

### 추가 응답 형식

```
✅ 팁 저장 완료
카테고리: 청소
팁: 전자레인지에 물+레몬 2분 돌리면 내부 찌든 냄새 제거됨
출처: (없음 or URL)
```

## Examples

**예시 1 — 질문**
입력: "수건에서 쉰내 나는데 어떻게 해?"
처리: `search_tips.py "수건 쉰내"` → 빨래 카테고리에서 매칭
출력:
```
[빨래] 수건 쉰내 제거
- 내용: 60도 이상 뜨거운 물에 베이킹소다 2큰술 + 식초 1컵 넣고 30분 담근 후 세탁
- 출처: (DB 기본)

💡 세탁기 통세척이 안 되어 있으면 수건이 아니라 세탁기 문제일 수도 있음.
```

**예시 2 — 팁 추가**
입력: "이거 좋더라: 후라이팬 기름때는 밀가루 뿌리고 닦으면 잘 빠짐"
처리: 카테고리 = 청소, 출처 없음 → `add_tip.py` 실행
출력:
```
✅ 팁 저장 완료
카테고리: 청소
팁: 후라이팬 기름때는 밀가루 뿌리고 닦으면 잘 빠짐
출처: (없음)
```

**예시 3 — 카테고리별 요청**
입력: "절약 팁 3개만"
처리: `search_tips.py --category 절약 --limit 3`
출력: 절약 카테고리에서 3개 나열

## Edge cases

- 빈 입력: "어떤 살림 팁이 궁금하세요?" 한 번 물어봄
- DB에 매칭 없음: 일반 지식으로 답하고 "DB에는 관련 팁 없음" 명시 + 저장할지 제안
- 출처가 URL이면 그대로 표시, 출처가 "엄마", "유튜브에서 봤음" 같은 자연어면 그대로 보존
- 카테고리 모호 (예: "음식물쓰레기 냄새" → 청소? 생활꿀팁?): 한 번만 확인
- 중복 팁: 같은 내용 80% 이상 일치하면 "이미 비슷한 팁이 있음" 알림 후 중복 저장 방지

## Limitations

- 키워드 매칭 기반이라 동의어/유의어는 Claude의 판단에 의존
- 외부 URL 검증 안 함 (사용자가 준 URL 그대로 저장)
- DB가 커지면 검색 속도 느려질 수 있음 (수천 개 단위 전엔 문제없음)
- 의학·약품·전기공사 같은 전문 영역 팁은 받지만 안전 보장 안 함
